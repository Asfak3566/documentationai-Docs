## Branding and Themes

Set up branding to reflect your project's identity. Use the dashboard at `https://dashboard.example.com/settings/branding` to upload a logo and define colors.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to **Account > Branding**.
  </Step>
  <Step title="Upload Logo">
    Drag and drop your SVG or PNG logo (max 512x512px).
  </Step>
  <Step title="Set Colors">
    Enter your primary color: `#3B82F6`.
  </Step>
  <Step title="Preview and Save">
    Preview changes across the site, then save.
  </Step>
</Steps>

<CodeGroup tabs="JSON,Env">
````json
{
  "brand": {
    "color": "#3B82F6",
    "logo": "https://your-site.com/logo.svg"
  }
}
````
````bash
BRAND_COLOR=#3B82F6
LOGO_URL=https://your-site.com/logo.svg
````
</CodeGroup>

## User Access and Permissions

Control access with role-based permissions. Define roles like Viewer, Editor, and Admin.

<Tabs>
  <Tab title="Viewer" icon="eye">
    Read-only access.
  </Tab>
  <Tab title="Editor" icon="edit-3">
    Edit and preview docs.
  </Tab>
  <Tab title="Admin" icon="shield">
    Full control, including user management.
  </Tab>
</Tabs>

Use these parameters in your API calls to `https://api.example.com/v1/permissions`:

<ParamField path="role" param-type="string" required="true">
  User role: `viewer`, `editor`, or `admin`.
</ParamField>

<ParamField path="permissions" param-type="array" required="false">
  Specific actions like `edit:pages`, `publish:drafts`.
</ParamField>

| Role    | Can Edit | Can Publish | Manage Users |
|---------|----------|-------------|--------------|
| Viewer  | No       | No          | No           |
| Editor  | Yes      | No          | No           |
| Admin   | Yes      | Yes         | Yes          |

## Workflows and Templates

Streamline content creation with custom workflows and templates.

<Expandable title="Custom Workflow Example" default-open="true">
  Create a workflow for reviews:

````json
{
  "workflow": {
    "steps": ["draft", "review", "approve"],
    "assignees": ["editor-team"]
  }
}
````
</Expandable>

<Callout kind="tip">
  Start with built-in templates for API docs or changelogs to accelerate setup.
</Callout>

## Notifications and Alerts

Enable notifications for doc updates via email, Slack, or webhooks.

Configure webhooks at `https://dashboard.example.com/settings/notifications`:

<ParamField header="X-Webhook-Signature" param-type="string" required="false">
  HMAC signature for verification.
</ParamField>

<ParamField body="event" param-type="string" required="true">
  Event type: `page.updated`, `user.invited`.
</ParamField>

<Callout kind="alert">
  Always validate webhook payloads to prevent unauthorized actions.
</Callout>

<Expandable title="Troubleshooting Notifications">
  - Check webhook URL accessibility.
  - Verify `YOUR_WEBHOOK_SECRET` matches.
  - Test with curl:

````bash
curl -X POST https://your-webhook-url.com/webhook \
  -H "Content-Type: application/json" \
  -d '{"event": "test"}'
````
</Expandable>
